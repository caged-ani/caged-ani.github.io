2.2 字符型数据与数据型数据之间的转换
char 型数据可以转换成任意一种数值类型，反之亦然
(1)整数 — char：只用到该数据的低十六位，其余部分忽略
char ch = (char)0XAB0041; // 0041 is assigned
(2)浮点值 — char：首先将浮点值转换成 int 型，然后再将这个整型值转换为 char 型
char ch = (char)65.25; // Decimal 65 is assigned, ch is A
(3)char — 数值型：这个字符的 Unicode 就被转换成某个指定的数值类型
int i = (int)'A'; // i is 65
这种隐式转换只能在转换结果适用于目标变量情况下(0 ~ FFFF的任何一个十六进制整数)；否则必须显式转换

3 String类型

String与System类和Scanner类一样，都是Java库中预定义的类。String类型不是基本类型，而是引用类型(reference type)，任何Java类都能作为引用类型来声明一个变量。 3.1 String对象的简单方法**(6个)

3.2 Java允许在不创建新变量的情况下，使用字符串字面值直接引用字符串
"Welcome to Java".length()是正确的

3.3 字符串连接
如果 i = 1, j = 2
System.out.println("i + j is " + i + j) // "i + j is "首先和 i 的值连接
输出为
i + j is 12
System.out.println("i + j is " + (i + j))
输出才是
i + j is 3

3.4 从控制台读取字符串
(1)调用Scanner 对象上的 next() 方法读取以空白字符结束的字符串
(2)用 nextLine() 方法读取一整行文本，以按下回车键为结束标志
(3)next()、nextByte()、nextShort()、nextInt()、nextLong()、nextFloat()和nextDouble()的输入称为基于标记的输入，因为它们读取采用空白字符分隔的单个元素，而不是整行。nextLine()方法称为基于行的输入
(4)为避免输入错误，程序中不要在基于标记的输入后使用基于行的输入

3.5 字符串比较(7个)

(2)compareTo 返回的值是依据 s1 和 s2从左到右第一个不同字符之间的距离得出的
